What Is a Variable?

Part 1
The Concept

A Labeled Jar

A variable is simply a container with a label on it. You give it a name, and you put a value inside. The label stays the same, but you can swap out what's inside.

x = 1             ← The container called "x" now holds the number 1
x = 2             ← We replaced what was inside. Now "x" holds 2.
name = "Sarah"      ← The container called "name" holds the word Sarah.

Setting vs. Checking

Part 2
Set ( = )

One Equals Sign

This means "put this value in the container."

  • x = 5 (Set x to 5)
  • status = "IN_PROGRESS"
Check ( == )

Two Equals Signs

This means "is this true or false?" Used for making decisions.

  • x == 2 (Is x equal to 2?)
  • x != 0 (Is x NOT equal to 0?)
  • x > 3 (Is x greater than 3?)

Key Rule: One = means SET. Two == means CHECK.

Types of Variables (1/2)

Part 3

Not everything is a number. Variables hold specific types of data:

1. Integer integer

A whole number. No decimals.

CANCEL_ATTEMPTS = 0

Use when: You're counting something.

2. Float float

A number with decimals.

TOTAL_WAIT_TIME = 14.5

Use when: You need precision (money, time).

3. String string

Text — a word, name, or sentence in quotes.

ROUTING_NUM = "021000021"

Use when: Storing text, emails, or IDs.

4. Text text

Same as string, but for long paragraphs.

AI_COPY = "☀️ Your summer..."

Use when: Content is multiple sentences.

Types of Variables (2/2)

Part 3

5. Boolean boolean

TRUE or FALSE. Like a light switch.

AUTHORIZATION = TRUE

Use when: Something is yes/no, on/off.

6. Enum enum

Restricted to a pre-defined list of options.

STATUS = "IN_PROGRESS"

Use when: There is a fixed list of valid choices (e.g. dropdown menu).

7. Timestamp timestamp

A specific date AND time, down to the second.

"2026-06-01T10:30:00Z"

Use when: You need exact moments.

8. Date date

Just the calendar date — no time attached.

EXPECTED_DUE = "2026-06-02"

Use when: You only care about the day.

Shorthand Operators

Part 4
+= and -=

The Shortcut

When you want to add or subtract from a variable's current value, there's a shortcut.

The Long WayShorthandWhat It Does
x = x + 1x += 1Add 1 to x
x = x - 1x -= 1Subtract 1 from x
x = x + 5x += 5Add 5 to x
Real Example

TRIAL_DESIGN_SUBMISSIONS = 3   // Starts with 3
(Client submits a request...)
TRIAL_DESIGN_SUBMISSIONS -= 1   // Now it's 2

Scenario Walkthrough

Part 5
Goal

Client hasn't responded in 2 days

If a client doesn't respond to a design review after 2 days, send a follow-up.

1. Designer submits design.
→ Set: n = 0 (Days without response counter)
→ Set: STATUS = "IN_REVIEW"

2. Day 1 passes. Did client respond?
IF YES: STATUS = "IN_PROGRESS" (Exit loop)
IF NO: n += 1 (n is now 1). Check: Is n >= 2? No. Wait another day.

3. Day 2 passes. Did client respond?
IF YES: STATUS = "IN_PROGRESS"
IF NO: n += 1 (n is now 2). Check: Is n >= 2? YES!
→ SEND follow-up message to client.

Key Takeaways

Review
  • n = 0 → We SET the counter at the start.
  • n += 1 → Each day without a response, we ADD 1.
  • n >= 2 → We CHECK if the counter has reached our threshold.
  • The Loop → The system keeps cycling daily until either the client responds OR the counter triggers the message.
